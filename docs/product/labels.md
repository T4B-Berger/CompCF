# CompCF Label Taxonomy

## Domain labels
`domain:identity`, `domain:organizer`, `domain:event-publication`, `domain:divisions-categories-pricing`, `domain:registration`, `domain:volunteer-recruitment`, `domain:volunteer-onboarding`, `domain:volunteer-scheduling`, `domain:judging-live`, `domain:care`, `domain:build-logistics`, `domain:competition-programming`, `domain:scheduling-heats`, `domain:scoring-leaderboard`, `domain:public`, `domain:admin-audit`, `domain:docs`, `domain:infra`

## Type labels
`type:epic`, `type:story`, `type:task`, `type:bug`, `type:spike`

## Priority labels
`priority:p0`, `priority:p1`, `priority:p2`

## Phase labels
`phase:mvp`, `phase:post-mvp`, `phase:later`

## Workflow/support labels
`blocked`, `needs-spec`, `ready-for-codex`, `good-first-pr`

## Notes
- Legacy labels should be minimized when redundant with this taxonomy.
- Keep labels intentionally small and execution-oriented.
