## Git and PR rules
- Ensure `origin` points to `https://github.com/T4B-Berger/CompCF.git` before any push.
- Use `npm ci` when `package-lock.json` is present.
- For GitHub write tests, create only one dedicated test file and no other changes.
- When asked to validate PR capability, create a branch, push it, and open a draft PR to `main`.
